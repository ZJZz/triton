把任意 Layout 转成 Linear Layout

可视化 python/tutorials/gluon/09-tma-gather-scatter.py 第 140–211 行的 to_linear_layout 工具 —— 它把 Gluon 的分布式 layout 翻译成统一的 DistributedLinearLayout， 用来判断一个 layout 是否能用于 async_gather / async_scatter 的 gather4/scatter4 指令。

① 什么是 Linear Layout？

一个 分布式 layout 回答的问题是：张量里的每个逻辑元素，被「哪一份硬件资源」持有？硬件坐标分四层（从小到大）：

register 寄存器（一个线程内的多个值） lane 通道（warp 内的 32 个线程） warp（CTA 内的若干 warp） block / CGA（多 CTA，本例为空）

Linear layout 的核心思想：这个「硬件坐标 → 张量下标」的映射，在 GF(2)（二进制按位异或） 意义下是线性的。 于是只需记录每一层「翻动某一个 bit 会让张量下标变化多少」，这个变化量就叫一个 basis（基）。

element_index = ⨁ // 按位异或 reg_bases[i] for each bit i set in register_id ⨁ lane_bases[i] for each bit i set in lane_id ⨁ warp_bases[i] for each bit i set in warp_id

例如 reg_bases=[[1],[2],...] 表示：寄存器号的 bit0 让下标 +1，bit1 让下标 +2 —— 所以 寄存器 0,1,2,3 恰好持有连续的元素 0,1,2,3。这正是 gather4「4 个连续行进连续寄存器」的要求。 而 lane_bases 全是 [0] 表示「翻动 lane 号下标不变」，即 同一 warp 内 32 个线程持有完全相同的数据（广播）。

教程结论（第 136–139 行）：一个 layout 对 gather4 合法 ⟺ 前两个 reg_bases 是 [1] 和 [2] 且 所有 lane_bases 都是 [0]。
