第一部分 · CGA 与 Multi-CTA Layout（cga_layout）

CGA = 谁和谁一组 cga_layout = 每个 CTA 拿哪块
CGA 是执行层面的硬件分组：把 ≤16 个 CTA 绑成一个 cluster 让它们协作； cga_layout 是数据层面的分片规则：一个把 CTA-id 的每个 bit 映射到张量坐标的线性映射。 在 中间面板切换 cga_layout，把鼠标移到张量分片或右侧 CTA 表上 → 三个视图联动。
① 布局层级里 CGA 在哪 从下往上：元素 → … → CGA
② cga_layout 把张量切给各 CTA
③ CTA-id → 坐标（线性映射） id 的每个 bit 配一个基
④ 连续 vs 交错 · 两种表示 cga_layout 永远是「连续分块」；交错切分要用 LinearEncoding
