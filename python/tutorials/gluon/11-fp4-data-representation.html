③ 两个除法分别"为什么"

// 2  打包 (packing) — 关于存储

fp4 只有 4 bit，而内存最小可寻址单位是 1 字节 (8 bit)。Gluon 没有原生 fp4 存储类型， 于是沿 K 维把相邻 2 个 fp4 塞进 1 个 uint8（高 4 位 + 低 4 位）。

storage_K = K / ELEM_PER_BYTE = K / (8 / bitwidth)

一个 uint8 = 两个 fp4 (e2m1 = 1 符号 + 2 指数 + 1 尾数)。 所以 fp4 → ELEM_PER_BYTE = 2；fp8 已经是 8 bit → ELEM_PER_BYTE = 1，不打包。

// VEC_SIZE  量化粒度 (scaling) — 关于精度

block scaling 沿 K 把元素切成等长的块，每块 VEC_SIZE 个元素共用 1 个 scale 因子。 反量化时 scale 被广播乘到这 VEC_SIZE 个元素上。所以一行需要的 scale 个数 = K 除以块长。

scale_K = K / VEC_SIZE
VEC_SIZE 不是随便取的，由硬件支持的量化方案固定： nvfp4 → 16（NVIDIA 专有，scale 用 float8_e4m3fn）； MX 格式 (mxfp4/mxfp8) → 32（scale 用 e8m0，即 8 指数位 0 尾数位的 2 的幂）。

④ 最终 shape（当前参数）

同一个张量有逻辑 shape（数学上的 M×K）和存储 shape（显存里 uint8 的实际形状），别混淆。
代码里对应：A_ELEM_PER_BYTE = 2 if A_IS_FP4 else 1； BLOCK_K = a_desc.block_type.shape[1] * A_ELEM_PER_BYTE（descriptor 存的是字节维，乘回去才是逻辑 K）； scale 的 K 偏移用 k // VEC_SIZE。

⑤ A / B 数据块：两种分组落在二维矩阵的哪里

把 K 轴铺成二维（行 = M 或 N，列 = K）。蓝深浅竖条是 scale 分组，每 VEC_SIZE 列一条 （蓝粗线 = // VEC_SIZE 边界）；琥珀细线是打包字节边界，每 2 列一根 （// 2，fp8 不打包则没有）。把鼠标悬停在右侧某个 scale 方块，左边会高亮它在原始数据里 影响的那一块 1×VEC_SIZE 区域；悬停某个元素，则同时高亮它所属的 scale 组和打包字节。
scale 组（交替深浅） // VEC_SIZE 边界 // 2 字节边界 悬停：scale 影响区 悬停：元素所在字节

每个 scale 只作用于自己这一行内连续 VEC_SIZE 个元素，不跨行、不跨 scale 组。 B operand 结构完全相同，只是行数是 N；区别仅在喂给 MMA 时 B 以转置 descriptor b_smem.permute((1,0)) 传入， 其 scale 形状仍是 [BLOCK_N, BLOCK_K//VEC_SIZE]。
