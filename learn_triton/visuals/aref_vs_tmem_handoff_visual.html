`NVWSInsertAref` 和 TMEM Ownership Handoff 的差异

两者都在做 partition 之间的 handoff，但 handoff 的“对象”和“协议重点”不一样。 `NVWSInsertAref` 主要处理 value / SMEM buffer availability；`InsertTmemAref` 处理的是 TMEM mutable state 的 active ownership transfer。

Aref value handoff TMEM ownership handoff 共同点 都用 aref + put/get + barrier lowering

并排看：差别到底大不大？

不算“完全不同机制”，因为两者最终都走 aref enter/exit + barrier lowering。 但差别也不小：普通 `InsertAref` 的核心问题是“把跨 partition 的 SSA/value 交给对方”； TMEM 路径的核心问题是“把一块可变状态和它的使用权交给对方”。

`NVWSInsertAref`

tensor / scalar / SMEM value availability producer -> consumer

它把“跨 partition 的直接 SSA/use-def 边”改写成“producer 把值写进 aref backing buffer， consumer 再从这个 buffer 读出来”。

before 直接跨 partition use

原来是 producer 结果直接被另一个 partition 的 use 消费。

producer aref.put.enter

选一个 aref buffer 视图，producer 在这里写入 value。

buffer SMEM backing buffer

值落到共享可访问的 buffer，而不是继续停留在 producer SSA/register。

consumer aref.get.enter

consumer 通过 indirection 取到 data buffer，再 local_load / unsplat / memdesc view。

after value becomes locally consumable

consumer 拿到的是“重新物化”的可消费 value，而不是共享 ownership。

Partition A

produce tensor / scalar / descriptor-load result

put into aref buffer
post full / later wait
Aref buffer (SMEM)

本质上是一个共享可见的中转存储槽位。

重点是：让另一个 partition 看到并读到这份值。

Partition B

get buffer, local_load / use rewritten value

get from indirection
release empty later
Consumer View

关键是 data availability，不是长期占有这块状态。

`InsertTmemAref` / TMEM Ownership Handoff

TMEM state mutable ownership ping-pong between two groups

它不是简单地“把值给 consumer”。它处理的是一块 TMEM mutable state： 谁现在有资格继续读/写这块状态、什么时候把 baton 交给另一个 partition。

before same TMEM used across partitions

不同 partition 会继续在同一块 TMEM 上做 store / load / mma。

owner A acquire / put.enter

A 先拿到这块 TMEM 的 active ownership，推进状态。

handoff put.exit -> full

不是仅仅说“数据好了”，而是 ownership baton 被交给 B。

owner B get.enter

B 在 wait full 成功后，成为这块 TMEM 的合法 active user。

return get.exit -> empty

B 用完后归还 slot，之后 A 才能继续复用或重占。

Partition A

producer owner / accumulator setup / TMEM write

handoff ownership
full signals B may proceed
Same TMEM mutable state

传递的不是独立复制值，而是“谁现在能继续操作这块状态”。

重点是 active ownership，不只是 visibility。

Partition B

consumer owner / mma consume / TMEM read

return ownership
empty re-enables reuse
Ring Reuse

后续 slot 复用取决于 ownership 被显式归还到 empty。

共同点：两者最后都依赖 aref.put/get.enter/exit 和 `NVWSLowerAref` 生成的 barrier/wait/fence/arrive 体系。
最大差异：`NVWSInsertAref` 的默认心智模型是 “move value availability”； `InsertTmemAref` 的默认心智模型是 “move mutable-state ownership”。
