Theme: TMA / TMEM boundary Question: 哪条路径在回答哪类问题 Source: compiler.py + pass sources

TMA 和 TMEM 会在同一个 kernel 里同时出现，但它们解决的不是同一个 contract

这页专门用来消除“为什么笔记里总感觉它们缠在一起”的问题。 先看 capability/pipeline 的交集，再看数据路径真正在哪一层分叉： TMA 负责 global <-> shared transport；TMEM 负责 Blackwell tensor-core state 的 placement / layout / allocation / synchronization。

Hopper 也有 TMA

sm90+ 走 tma_lowering。所以 “看到 TMA” 不等于 “已经进入 TMEM 路径”。

Blackwell 才有这条 TMEM 主线

sm100+ 才会走 promote_lhs_to_tmem、allocate_tensor_memory、tmem_barrier_insertion 这套 accumulator 常驻 TMEM 的路径。

相遇点

两条线会在 shared/TMEM alias 和 proxy visibility 上相互影响，所以 proxy_fence_insertion 常常是读懂边界的关键。

  • 如果问题是 swizzle / descriptor / tensormap / barrier_expect，先站到 TMA 视角。
