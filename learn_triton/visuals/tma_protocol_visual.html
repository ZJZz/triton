TMA 不是一条 copy 指令，而是一套 transport + barrier + proxy contract

这个页面不把注意力放在 pass 名字列表，而是把 tt.descriptor_* 为什么会 lower 成一串 op 画成协议时间线。重点是三件事：shared 目的地先建立、mbarrier 完成关系显式化、generic/async proxy 可见性单独补齐。

你真正要记的 invariant

descriptor op 不直接“产出 tensor”。load 路径先把 tile 异步写到 shared，再通过 barrier 和 local_load 把它变成寄存器值。

为什么 store 也不只是 store

shared 上的 generic write 必须先经过 fence_async_shared，async TMA store engine 才能把这块 shared 看成合法输入。

为什么还有 proxy fence

wait_barrier 管 completion，不自动解决 generic proxy 和 async proxy 的跨域可见性；这部分由 ProxyFenceInsertion 单独负责。

切换不同 descriptor 语义，观察它在 lowering 后分别建立了哪类 contract。
